Markdown

Sección 1: Introducción

  • Markdown (.md) es la estandar para documentación y ahora hasta para páginas o sitios estaticos. Como en HTML el documento principal es index.html, en markdown es README.md
  • Para comentar en markdown es igual que en HTML <!-- comentario -->

Sección 2: Parrafos, cursiva, negritas y tachado

  • Los parrafos son solo texto plano en md, sin etiquetas ni nada
  • Las cursivas se escriben con guiones bajos a los lados
    • _Esto es una cursiva enn md_
  • Las negritas se escriben con 2 asteriscos a los lados
    • **Esto es una negrita enn md**
  • Los textos tachados se escriben con 2 virguilillas (~) a los lados
    • ~~Esto es un texto tachado md~~

Sección 3: Encabezados

  • Los encabezados son como los h1en HTMl, y se escriben con el signo de numeral (#), dependiendo del nivel, se agrega otro, son 6 niveles, osea hasta 6 numerales
    • # Encabezado 1
## Encabezado 2
### Encabezado 3

Sección 4: Enlaces

  • Un enlace normal es con corchetes [] el nombre del link, y el link en parentesis ()
    • [Youtube](https://youtube.com)
  • Para los enlaces internos o anclas, se utilizan los encabezados, por ejemplo el encabezado ## hola mundo y quiero ir hacia el, pondria el link [link interno](#hola-mundo) markdown los espacios los lee como guiones, y borra el espacio inicial, y no importa el nivel del encabezado, en el ancla se utiliza un numeral (#), y si estas utilizando Visual Studio Code en los parentesis te va a autocompletar los encabezados para que lo seleccione

Sección 5: Imagenes

  • Las imagenes se escriben como los enlaces solo que se les antepone un signo de admiracion de cierre (!)
    • ![hola](https://link-imagen.png)
  • Puede ser un link de internet, o si esta dentro del proyecto, la ruta
    • ![hola-ruta](img/imagen.png)

Sección 6: Divisiones

  • Para hacer lineas divisoras, se utilizan 3 guines medios ---, eso será como un <hr> en HTML

Sección 7: Listas

  • Listas ordenadas
  • Se ponen 1. en cada linea, pero el interprete pondra la lista ordenada (1,2,3,4,5), la buena practica seria poner solo 1. en cada elemento de la lista, porque tambien lee si se pone 1. 2. 3. pero es mejor solo 1. que lo interpreta igual
    • 1. yoiner
1. duran
1. rios
  • Listas desordenadas
  • Se pueden hacer con asteriscos (*) o guiones (-)
    • - yoiner
- duran
- rios
  • Sublistas
  • Las sublistas se hacen con tabulaciones
    • 1. yoiner
    1. duran
    1. re pro
        1. rios pro
  • En la visualizacion se nota que Markdown va haciento las tabulaciones, en las listas desordenadas hasta cambia el estilo de las viñetas sublistas

Sección 8: Citas

  • Por ejemplo en HTML se utiliza la etiqueta <cite> para citar, el markdow se utiliza el mayor que (>)
    • > hola mundo, cita en linea
  • Para una cita en bloque, se utiliza varios (>)
    • > cita
> en
> bloque

Sección 9: Tablas

  • Utilizamos el signo pleca (|) para crear las tablas, para decirle a markdown que vamos a crear una tabla, despues de la primera fila, creamos otra fila con solo guiones
    • |nombre|apellido|numero|
|---|---|---|
|yoiner|duran|12345678|

Sección 10: Codigos

  • Para escribir codigo en linea, utilizamos el acento grave (``)
    • `console.log("hola mundo")`
  • Para escribir codigo en bloque utilizamos 3 acentos graves, y utilizando este modo, podemos decirle que lenguaje es y ayudara pintando la sintaxis, al inicio pegado al acento grave se pone la extension del lenguaje a escribir
    • ```py
def sumar(a+b):
  return a+b
```

Sección 10: Mostrar codigo de Markdown

  • Se antepone un (\) en cada signo/sintaxis de markdown
    • \*\*negrita - mostrar codigo\*\*

\_cursiva\_

\# encabezado
